## 🔹 Types of SQL Commands

SQL commands are mainly divided into **5 types**:

---

## 1️⃣ DDL (Data Definition Language)

👉 Used to **define or modify database structure**

**Commands:**

* `CREATE` → Create database/table
* `ALTER` → Modify table structure
* `DROP` → Delete table/database
* `TRUNCATE` → Remove all records (faster than DELETE)

**Example:**

```sql
CREATE TABLE Students(id INT, name VARCHAR(50));
ALTER TABLE Students ADD age INT;
DROP TABLE Students;
```

---

## 2️⃣ DML (Data Manipulation Language)

👉 Used to **manipulate data (records)**

**Commands:**

* `INSERT` → Add new data
* `UPDATE` → Modify existing data
* `DELETE` → Remove data

**Example:**

```sql
INSERT INTO Students VALUES (1, 'Avi');
UPDATE Students SET name='Rahul' WHERE id=1;
DELETE FROM Students WHERE id=1;
```

---

## 3️⃣ DQL (Data Query Language)

👉 Used to **retrieve data**

**Command:**

* `SELECT` → Fetch data from database

**Example:**

```sql
SELECT * FROM Students;
```

---

## 4️⃣ DCL (Data Control Language)

👉 Used to **control access/permissions**

**Commands:**

* `GRANT` → Give permission
* `REVOKE` → Remove permission

**Example:**

```sql
GRANT SELECT ON Students TO user1;
REVOKE SELECT ON Students FROM user1;
```

---

## 5️⃣ TCL (Transaction Control Language)

👉 Used to **manage transactions**

**Commands:**

* `COMMIT` → Save changes permanently
* `ROLLBACK` → Undo changes
* `SAVEPOINT` → Temporary checkpoint

**Example:**

```sql
SAVEPOINT sp1;
ROLLBACK TO sp1;
COMMIT;
```

---

## 🔥 Final Quick Summary

| Type | Full Form           | Purpose               |
| ---- | ------------------- | --------------------- |
| DDL  | Data Definition     | Structure (table, DB) |
| DML  | Data Manipulation   | Data change           |
| DQL  | Data Query          | Fetch data            |
| DCL  | Data Control        | Permissions           |
| TCL  | Transaction Control | Manage transactions   |

---

## ✅ Placement Tip

* **SELECT = DQL (most used in interviews)**
* **INSERT/UPDATE/DELETE = DML**
* **CREATE/ALTER/DROP = DDL**

# 📊 SQL Aggregate Functions (Core Concept)

👉 Perform calculation on **multiple rows → return single value**

### 🔹 Key Points

* Used with **GROUP BY**
* Ignore **NULL values** *(except `COUNT(*)`)*
* Output = **single value per group**

---

## 🔹 Aggregate Functions Table

| Function  | Description    | Example                             |
| --------- | -------------- | ----------------------------------- |
| `MIN()`   | Smallest value | `SELECT MIN(Price) FROM Products;`  |
| `MAX()`   | Largest value  | `SELECT MAX(Price) FROM Products;`  |
| `COUNT()` | Number of rows | `SELECT COUNT(*) FROM Products;`    |
| `SUM()`   | Total sum      | `SELECT SUM(Quantity) FROM Orders;` |
| `AVG()`   | Average value  | `SELECT AVG(Price) FROM Products;`  |

---

## 🔹 Important Concepts

### 1️⃣ MIN() / MAX()

* Works on **number, string, date**

```sql
SELECT MIN(Price) AS SmallestPrice FROM Products;
SELECT MAX(Price) FROM Products;
```

👉 With GROUP BY:

```sql
SELECT MAX(Price), CategoryID
FROM Products
GROUP BY CategoryID;
```

---

### 2️⃣ COUNT()

| Type                  | Meaning                   |
| --------------------- | ------------------------- |
| `COUNT(*)`            | All rows (including NULL) |
| `COUNT(column)`       | Non-null values           |
| `COUNT(DISTINCT col)` | Unique non-null values    |

```sql
SELECT COUNT(*) FROM Products;
SELECT COUNT(ProductID) FROM Products WHERE Price > 20;
```

---

### 3️⃣ SUM()

* Only for **numeric columns**

```sql
SELECT SUM(Quantity) FROM OrderDetails;
```

👉 With expression:

```sql
SELECT SUM(Quantity * 10) FROM OrderDetails;
```

---

### 4️⃣ AVG()

```sql
SELECT AVG(Price) FROM Products;
```

---

# 🔍 SQL LIKE Operator (Pattern Matching)

👉 Used in `WHERE` for **searching patterns**

## 🔹 Wildcards

| Symbol | Meaning              |
| ------ | -------------------- |
| `%`    | 0 or more characters |
| `_`    | Exactly 1 character  |

---

## 🔹 Common Patterns

| Use Case                      | Query         |
| ----------------------------- | ------------- |
| Starts with 'A'               | `LIKE 'A%'`   |
| Ends with 'a'                 | `LIKE '%a'`   |
| Contains 'or'                 | `LIKE '%or%'` |
| Second letter 'r'             | `LIKE '_r%'`  |
| At least 3 chars starting 'a' | `LIKE 'a__%'` |

```sql
SELECT * FROM Customers WHERE Name LIKE 'A%';
```

---

# 🔹 SQL IN Operator

👉 Shortcut for multiple `OR` conditions

```sql
SELECT * FROM Customers
WHERE Country IN ('India','USA','UK');
```

### NOT IN

```sql
WHERE Country NOT IN ('India','USA');
```

### IN with Subquery

```sql
SELECT * FROM Customers
WHERE CustomerID IN (SELECT CustomerID FROM Orders);
```

---

# 🔹 SQL BETWEEN Operator

👉 Used for **range (inclusive)**

```sql
SELECT * FROM Products
WHERE Price BETWEEN 10 AND 20;
```

### NOT BETWEEN

```sql
WHERE Price NOT BETWEEN 10 AND 20;
```

---

# 🔹 SQL Aliases (AS)

👉 Temporary name for column/table

## Column Alias

```sql
SELECT CustomerName AS Name FROM Customers;
```

## Table Alias

```sql
SELECT c.CustomerName
FROM Customers AS c;
```

---

# 🔥 Final Quick Revision

* **Aggregate = multiple → single value**
* **COUNT(*) includes NULL, others ignore NULL**
* **LIKE = pattern search (`%`, `_`)**
* **IN = multiple OR shortcut**
* **BETWEEN = range (inclusive)**
* **Alias = temporary name using AS**

# JOINS

## 1. INNER JOIN

* Returns only matching rows from both tables.
* `JOIN` = `INNER JOIN`

```sql id="9pb0h4"
SELECT *
FROM A
INNER JOIN B
ON A.id = B.id;
```

✅ Match in both tables required.

---

## 2. LEFT JOIN

* Returns all rows from left table.
* Matching rows from right table.
* No match → `NULL`

```sql id="ld0r4q"
SELECT *
FROM A
LEFT JOIN B
ON A.id = B.id;
```

✅ Left table fully preserved.

---

## 3. RIGHT JOIN

* Returns all rows from right table.
* Matching rows from left table.
* No match → `NULL`

```sql id="3q1mow"
SELECT *
FROM A
RIGHT JOIN B
ON A.id = B.id;
```

✅ Right table fully preserved.

---

## 4. FULL JOIN

* Returns all rows from both tables.
* Matched + unmatched rows.

```sql id="0h5br8"
SELECT *
FROM A
FULL JOIN B
ON A.id = B.id;
```

✅ All data from both tables.

---

## 5. SELF JOIN

* Table joined with itself.

```sql id="5p2h1k"
SELECT *
FROM Employee E1, Employee E2
WHERE E1.manager_id = E2.emp_id;
```

✅ Used for hierarchy/manager relation.

---

# JOIN MEMORY TRICK

| Join Type | Meaning                  |
| --------- | ------------------------ |
| INNER     | Common rows only         |
| LEFT      | All left + matched right |
| RIGHT     | All right + matched left |
| FULL      | Everything from both     |

---

# UNION

## UNION

* Combines rows vertically.
* Removes duplicates.

```sql id="u3j9df"
SELECT name FROM A
UNION
SELECT name FROM B;
```

---

## UNION ALL

* Combines rows vertically.
* Keeps duplicates.

```sql id="nh8w2z"
SELECT name FROM A
UNION ALL
SELECT name FROM B;
```

---

# UNION RULES

✅ Same number of columns
✅ Similar data types
✅ Same order of columns

---

# MEMORY TRICK

| Operation | Direction  |
| --------- | ---------- |
| UNION     | Up-Down    |
| JOIN      | Left-Right |

---

# GROUP BY

* Makes groups of same values.
* Used with aggregate functions.

Functions:

* `COUNT()`
* `SUM()`
* `AVG()`
* `MAX()`
* `MIN()`

```sql id="pc7d3n"
SELECT department, COUNT(*)
FROM Employees
GROUP BY department;
```

✅ Groups rows with same department.

---

# HAVING

* Filters groups after `GROUP BY`.

```sql id="ev4u8m"
SELECT department, COUNT(*)
FROM Employees
GROUP BY department
HAVING COUNT(*) > 2;
```

---

# WHERE vs HAVING

| WHERE                 | HAVING              |
| --------------------- | ------------------- |
| Filters rows          | Filters groups      |
| Before grouping       | After grouping      |
| No aggregate normally | Used with aggregate |

---

# EXISTS

* Checks whether subquery returns rows.

```sql id="n1c7qf"
SELECT name
FROM Customers c
WHERE EXISTS (
    SELECT *
    FROM Orders o
    WHERE c.id = o.customer_id
);
```

✅ TRUE if at least one row exists.

---

# ANY

* Condition true for at least one value.

```sql id="4g5szq"
SELECT *
FROM Products
WHERE price > ANY (
    SELECT price
    FROM Expensive_Items
);
```

✅ At least one match required.

---

# ALL

* Condition true for every value.

```sql id="r8h2wu"
SELECT *
FROM Products
WHERE price > ALL (
    SELECT price
    FROM Expensive_Items
);
```

✅ Must satisfy all values.

---

# ANY vs ALL

| ANY          | ALL         |
| ------------ | ----------- |
| At least one | Every value |
| OR logic     | AND logic   |

---

# QUICK INTERVIEW POINTS

## Difference: UNION vs UNION ALL

| UNION              | UNION ALL        |
| ------------------ | ---------------- |
| Removes duplicates | Keeps duplicates |
| Slower             | Faster           |

---

## Difference: INNER vs LEFT JOIN

| INNER JOIN        | LEFT JOIN       |
| ----------------- | --------------- |
| Only matched rows | All left rows   |
| No match removed  | No match = NULL |

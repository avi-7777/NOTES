## 🔹 Types of SQL Commands

SQL commands are mainly divided into **5 types**:

---

## 1️⃣ DDL (Data Definition Language)

👉 Used to **define or modify database structure**

**Commands:**

* `CREATE` → Create database/table
* `ALTER` → Modify table structure
* `DROP` → Delete table/database
* `TRUNCATE` → Remove all records (faster than DELETE)

**Example:**

```sql
CREATE TABLE Students(id INT, name VARCHAR(50));
ALTER TABLE Students ADD age INT;
DROP TABLE Students;
```

---

## 2️⃣ DML (Data Manipulation Language)

👉 Used to **manipulate data (records)**

**Commands:**

* `INSERT` → Add new data
* `UPDATE` → Modify existing data
* `DELETE` → Remove data

**Example:**

```sql
INSERT INTO Students VALUES (1, 'Avi');
UPDATE Students SET name='Rahul' WHERE id=1;
DELETE FROM Students WHERE id=1;
```

---

## 3️⃣ DQL (Data Query Language)

👉 Used to **retrieve data**

**Command:**

* `SELECT` → Fetch data from database

**Example:**

```sql
SELECT * FROM Students;
```

---

## 4️⃣ DCL (Data Control Language)

👉 Used to **control access/permissions**

**Commands:**

* `GRANT` → Give permission
* `REVOKE` → Remove permission

**Example:**

```sql
GRANT SELECT ON Students TO user1;
REVOKE SELECT ON Students FROM user1;
```

---

## 5️⃣ TCL (Transaction Control Language)

👉 Used to **manage transactions**

**Commands:**

* `COMMIT` → Save changes permanently
* `ROLLBACK` → Undo changes
* `SAVEPOINT` → Temporary checkpoint

**Example:**

```sql
SAVEPOINT sp1;
ROLLBACK TO sp1;
COMMIT;
```

---

## 🔥 Final Quick Summary

| Type | Full Form           | Purpose               |
| ---- | ------------------- | --------------------- |
| DDL  | Data Definition     | Structure (table, DB) |
| DML  | Data Manipulation   | Data change           |
| DQL  | Data Query          | Fetch data            |
| DCL  | Data Control        | Permissions           |
| TCL  | Transaction Control | Manage transactions   |

---

## ✅ Placement Tip

* **SELECT = DQL (most used in interviews)**
* **INSERT/UPDATE/DELETE = DML**
* **CREATE/ALTER/DROP = DDL**

## 🔹 Introduction to SQL

* **SQL (Structured Query Language)** → Used to access & manipulate databases
* Standardized by:

  * **ANSI (1986)**
  * **ISO (1987)**
* Works with **RDBMS systems**

---

## 🔹 RDBMS (Relational Database Management System)

* Data stored in **tables (rows + columns)**
* Examples:

  * MySQL
  * Oracle
  * Microsoft SQL Server
* SQL is based on RDBMS

---

## 🔹 SQL Basics

* SQL keywords are **NOT case-sensitive**

  * `SELECT = select`

---

## 🔹 Important SQL Commands

| Command     | Purpose               |
| ----------- | --------------------- |
| SELECT      | Retrieve data         |
| INSERT INTO | Insert data           |
| UPDATE      | Modify data           |
| DELETE      | Remove data           |
| CREATE      | Create DB/table       |
| ALTER       | Modify structure      |
| DROP        | Delete table/database |
| INDEX       | Improve search speed  |

---

## 🔹 SELECT Statement

```sql
SELECT column1, column2 FROM table_name;
SELECT * FROM table_name;
```

---

## 🔹 DISTINCT (Unique Values)

```sql
SELECT DISTINCT column FROM table;
SELECT COUNT(DISTINCT column) FROM table;
```

---

## 🔹 WHERE Clause (Filtering)

```sql
SELECT * FROM table WHERE condition;
```

### Operators:

* `=` Equal
* `>` `<`
* `>=` `<=`
* `<>` or `!=` Not equal
* `BETWEEN` Range
* `LIKE` Pattern
* `IN` Multiple values

---

## 🔹 Pattern Matching

```sql
WHERE name LIKE 'A%'   -- starts with A
WHERE name LIKE '_a%'  -- second letter 'a'
```

---

## 🔹 ORDER BY (Sorting)

```sql
SELECT * FROM table ORDER BY column ASC;
SELECT * FROM table ORDER BY column DESC;
```

---

## 🔹 AND, OR, NOT

```sql
WHERE cond1 AND cond2   -- both true
WHERE cond1 OR cond2    -- any true
WHERE NOT condition     -- reverse result
```

---

## 🔹 NOT LIKE

```sql
SELECT * FROM table WHERE name NOT LIKE 'A%';
```

---

## 🔹 INSERT INTO

```sql
INSERT INTO table(col1, col2) VALUES(val1, val2);

-- OR
INSERT INTO table VALUES(val1, val2);
```

---

## 🔹 NULL Values

* NULL = **missing/unknown value**
* Not equal to `0` or `''`

### Check NULL:

```sql
WHERE column IS NULL;
WHERE column IS NOT NULL;
```

---

## 🔹 UPDATE

```sql
UPDATE table
SET column = value
WHERE condition;
```

⚠️ Without WHERE → updates ALL rows

---

## 🔹 DELETE

```sql
DELETE FROM table WHERE condition;
DELETE FROM table;   -- deletes all rows
```

---

## 🔹 DROP TABLE

```sql
DROP TABLE table_name;
```

---

## 🔹 LIMIT / TOP (Restrict Rows)

### MySQL:

```sql
SELECT * FROM table LIMIT 3;
```

### SQL Server:

```sql
SELECT TOP 3 * FROM table;
```

### Oracle:

```sql
SELECT * FROM table FETCH FIRST 3 ROWS ONLY;
```

---

## 🔹 TOP PERCENT

```sql
SELECT TOP 50 PERCENT * FROM table;
```

---

* Use **DISTINCT** for uniqueness
* Use **LIKE** for pattern questions (very common in interviews)
* Remember:

  * `NULL ≠ 0`
  * `IS NULL` (not `=`)

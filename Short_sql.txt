# 🚀 SQL Placement Notes (Quick Revision)

## 🔹 What is SQL?

* SQL (Structured Query Language) → used to **store, retrieve, manipulate data**
* Works with **RDBMS (Relational Database Management System)**
* Standardized by **ANSI (1986)** and **ISO (1987)**
* Case-insensitive → `select = SELECT`

---

## 🔹 RDBMS Basics

* Data stored in **tables**
* Table = **rows (records) + columns (fields)**
* Examples: MySQL, Oracle, SQL Server

---

## 🔹 Most Important SQL Commands (Must Know)

| Command     | Use              |
| ----------- | ---------------- |
| SELECT      | Read data        |
| INSERT INTO | Add data         |
| UPDATE      | Modify data      |
| DELETE      | Remove data      |
| CREATE      | Create DB/Table  |
| ALTER       | Modify structure |
| DROP        | Delete table/db  |
| INDEX       | Improve search   |

---

## 🔹 SELECT (Most Asked 🔥)

```sql
SELECT column1, column2 FROM table_name;
SELECT * FROM table_name;
```

👉 Used to fetch data

---

## 🔹 DISTINCT (Unique Values)

```sql
SELECT DISTINCT column FROM table_name;
```

👉 Removes duplicates

---

## 🔹 COUNT + DISTINCT (Interview Favorite ⭐)

```sql
SELECT COUNT(DISTINCT column) FROM table_name;
```

---

## 🔹 WHERE (Filtering Data)

```sql
SELECT * FROM table_name WHERE condition;
```

### Operators:

* `=` Equal
* `>` `<` `>=` `<=`
* `<>` or `!=` Not equal
* `BETWEEN` Range
* `LIKE` Pattern
* `IN` Multiple values

---

## 🔹 Important Examples

```sql
SELECT * FROM Products WHERE Price BETWEEN 50 AND 60;

SELECT * FROM Customers WHERE City LIKE 'S%';
```

---

## 🔹 ORDER BY (Sorting)

```sql
SELECT * FROM table_name ORDER BY column ASC;
SELECT * FROM table_name ORDER BY column DESC;
```

👉 Default = ASC

---

## 🔹 AND / OR / NOT (Conditions)

### AND → All conditions true

```sql
SELECT * FROM Customers
WHERE Country='India' AND Name LIKE 'A%';
```

### OR → Any condition true

```sql
SELECT * FROM Customers
WHERE City='Mumbai' OR City='Pune';
```

### NOT → Reverse condition

```sql
SELECT * FROM Customers
WHERE NOT Country='India';
```

---

## 🔹 LIKE & NOT LIKE (Pattern)

* `%` → multiple characters
* `_` → single character

```sql
SELECT * FROM Customers WHERE Name LIKE 'A%';
SELECT * FROM Customers WHERE Name NOT LIKE 'A%';
```

---

## 🔹 INSERT INTO (Add Data)

### Method 1 (Best Practice ✅)

```sql
INSERT INTO table_name (col1, col2)
VALUES ('value1', 'value2');
```

### Method 2

```sql
INSERT INTO table_name
VALUES (value1, value2);
```

---

## 🔹 NULL (Very Important ⚠️)

* NULL = **missing/unknown value**
* Not equal to `0` or `''`

### ❌ Wrong:

```sql
WHERE column = NULL
```

### ✅ Correct:

```sql
WHERE column IS NULL
WHERE column IS NOT NULL
```

---

## 🔹 UPDATE (Very Important ⚠️)

```sql
UPDATE table_name
SET column1 = value1, column2 = value2
WHERE condition;
```

👉 Without `WHERE` → updates **all rows** (common mistake 🔥)

---

## 🔹 Key Interview Tips 💡

* Always use `WHERE` in UPDATE/DELETE
* Use `DISTINCT` to remove duplicates
* Use `COUNT(DISTINCT)` for unique count
* NULL cannot be compared using `=`
* `LIKE 'A%'` → starts with A
* `LIKE '%A'` → ends with A
* `LIKE '%A%'` → contains A

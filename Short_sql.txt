## 🔹 Types of SQL Commands

SQL commands are mainly divided into **5 types**:

---

## 1️⃣ DDL (Data Definition Language)

👉 Used to **define or modify database structure**

**Commands:**

* `CREATE` → Create database/table
* `ALTER` → Modify table structure
* `DROP` → Delete table/database
* `TRUNCATE` → Remove all records (faster than DELETE)

**Example:**

```sql
CREATE TABLE Students(id INT, name VARCHAR(50));
ALTER TABLE Students ADD age INT;
DROP TABLE Students;
```

---

## 2️⃣ DML (Data Manipulation Language)

👉 Used to **manipulate data (records)**

**Commands:**

* `INSERT` → Add new data
* `UPDATE` → Modify existing data
* `DELETE` → Remove data

**Example:**

```sql
INSERT INTO Students VALUES (1, 'Avi');
UPDATE Students SET name='Rahul' WHERE id=1;
DELETE FROM Students WHERE id=1;
```

---

## 3️⃣ DQL (Data Query Language)

👉 Used to **retrieve data**

**Command:**

* `SELECT` → Fetch data from database

**Example:**

```sql
SELECT * FROM Students;
```

---

## 4️⃣ DCL (Data Control Language)

👉 Used to **control access/permissions**

**Commands:**

* `GRANT` → Give permission
* `REVOKE` → Remove permission

**Example:**

```sql
GRANT SELECT ON Students TO user1;
REVOKE SELECT ON Students FROM user1;
```

---

## 5️⃣ TCL (Transaction Control Language)

👉 Used to **manage transactions**

**Commands:**

* `COMMIT` → Save changes permanently
* `ROLLBACK` → Undo changes
* `SAVEPOINT` → Temporary checkpoint

**Example:**

```sql
SAVEPOINT sp1;
ROLLBACK TO sp1;
COMMIT;
```

---

## 🔥 Final Quick Summary

| Type | Full Form           | Purpose               |
| ---- | ------------------- | --------------------- |
| DDL  | Data Definition     | Structure (table, DB) |
| DML  | Data Manipulation   | Data change           |
| DQL  | Data Query          | Fetch data            |
| DCL  | Data Control        | Permissions           |
| TCL  | Transaction Control | Manage transactions   |

---

## ✅ Placement Tip

* **SELECT = DQL (most used in interviews)**
* **INSERT/UPDATE/DELETE = DML**
* **CREATE/ALTER/DROP = DDL**

* Data stored in **tables (rows + columns)**

  * Oracle
  * Microsoft SQL Server
* SQL is based on RDBMS

* SQL keywords are **NOT case-sensitive**

  * `SELECT = select`

## 🔹 Important SQL Commands

| Command     | Purpose               |
| ----------- | --------------------- |
| SELECT      | Retrieve data         |
| INSERT INTO | Insert data           |
| UPDATE      | Modify data           |

| CREATE      | Create DB/table       |

## 🔹 SELECT Statement

```sql
SELECT column1, column2 FROM table_name;
SELECT * FROM table_name;
```

---

## 🔹 DISTINCT (Unique Values)

```sql
SELECT DISTINCT column FROM table;
SELECT COUNT(DISTINCT column) FROM table;
```

---

## 🔹 WHERE Clause (Filtering)

```sql
SELECT * FROM table WHERE condition;
```

### Operators:

* `=` Equal
* `>` `<`
* `>=` `<=`
* `<>` or `!=` Not equal
* `BETWEEN` Range
* `LIKE` Pattern
* `IN` Multiple values

---

## 🔹 Pattern Matching

```sql
WHERE name LIKE 'A%'   -- starts with A
WHERE name LIKE '_a%'  -- second letter 'a'
```

---

## 🔹 ORDER BY (Sorting)

```sql
SELECT * FROM table ORDER BY column ASC;
SELECT * FROM table ORDER BY column DESC;
```

---

## 🔹 AND, OR, NOT

```sql
WHERE cond1 AND cond2   -- both true
WHERE cond1 OR cond2    -- any true
WHERE NOT condition     -- reverse result
```

---

## 🔹 NOT LIKE

```sql
SELECT * FROM table WHERE name NOT LIKE 'A%';
```

---

## 🔹 INSERT INTO

```sql
INSERT INTO table(col1, col2) VALUES(val1, val2);

-- OR
INSERT INTO table VALUES(val1, val2);
```

---

## 🔹 NULL Values

* NULL = **missing/unknown value**
* Not equal to `0` or `''`

### Check NULL:

```sql
WHERE column IS NULL;
WHERE column IS NOT NULL;
```

---

## 🔹 UPDATE

```sql
UPDATE table
SET column = value
WHERE condition;
```

⚠️ Without WHERE → updates ALL rows

---

## 🔹 DELETE

```sql
DELETE FROM table WHERE condition;
DELETE FROM table;   -- deletes all rows
```

---

## 🔹 DROP TABLE

```sql
DROP TABLE table_name;
```

---

## 🔹 LIMIT / TOP (Restrict Rows)

### MySQL:

```sql
SELECT * FROM table LIMIT 3;
```

### SQL Server:

```sql
SELECT TOP 3 * FROM table;
```

### Oracle:

```sql
SELECT * FROM table FETCH FIRST 3 ROWS ONLY;
```

---

## 🔹 TOP PERCENT

```sql
SELECT TOP 50 PERCENT * FROM table;
```

# 📊 SQL Aggregate Functions (Core Concept)

👉 Perform calculation on **multiple rows → return single value**

### 🔹 Key Points

* Used with **GROUP BY**
* Ignore **NULL values** *(except `COUNT(*)`)*
* Output = **single value per group**

---

## 🔹 Aggregate Functions Table

| Function  | Description    | Example                             |
| --------- | -------------- | ----------------------------------- |
| `MIN()`   | Smallest value | `SELECT MIN(Price) FROM Products;`  |
| `MAX()`   | Largest value  | `SELECT MAX(Price) FROM Products;`  |
| `COUNT()` | Number of rows | `SELECT COUNT(*) FROM Products;`    |
| `SUM()`   | Total sum      | `SELECT SUM(Quantity) FROM Orders;` |
| `AVG()`   | Average value  | `SELECT AVG(Price) FROM Products;`  |

---

## 🔹 Important Concepts

### 1️⃣ MIN() / MAX()

* Works on **number, string, date**

```sql
SELECT MIN(Price) AS SmallestPrice FROM Products;
SELECT MAX(Price) FROM Products;
```

👉 With GROUP BY:

```sql
SELECT MAX(Price), CategoryID
FROM Products
GROUP BY CategoryID;
```

---

### 2️⃣ COUNT()

| Type                  | Meaning                   |
| --------------------- | ------------------------- |
| `COUNT(*)`            | All rows (including NULL) |
| `COUNT(column)`       | Non-null values           |
| `COUNT(DISTINCT col)` | Unique non-null values    |

```sql
SELECT COUNT(*) FROM Products;
SELECT COUNT(ProductID) FROM Products WHERE Price > 20;
```

---

### 3️⃣ SUM()

* Only for **numeric columns**

```sql
SELECT SUM(Quantity) FROM OrderDetails;
```

👉 With expression:

```sql
SELECT SUM(Quantity * 10) FROM OrderDetails;
```

---

### 4️⃣ AVG()

```sql
SELECT AVG(Price) FROM Products;
```

---

# 🔍 SQL LIKE Operator (Pattern Matching)

👉 Used in `WHERE` for **searching patterns**

## 🔹 Wildcards

| Symbol | Meaning              |
| ------ | -------------------- |
| `%`    | 0 or more characters |
| `_`    | Exactly 1 character  |

---

## 🔹 Common Patterns

| Use Case                      | Query         |
| ----------------------------- | ------------- |
| Starts with 'A'               | `LIKE 'A%'`   |
| Ends with 'a'                 | `LIKE '%a'`   |
| Contains 'or'                 | `LIKE '%or%'` |
| Second letter 'r'             | `LIKE '_r%'`  |
| At least 3 chars starting 'a' | `LIKE 'a__%'` |

```sql
SELECT * FROM Customers WHERE Name LIKE 'A%';
```

---

# 🔹 SQL IN Operator

👉 Shortcut for multiple `OR` conditions

```sql
SELECT * FROM Customers
WHERE Country IN ('India','USA','UK');
```

### NOT IN

```sql
WHERE Country NOT IN ('India','USA');
```

### IN with Subquery

```sql
SELECT * FROM Customers
WHERE CustomerID IN (SELECT CustomerID FROM Orders);
```

---

# 🔹 SQL BETWEEN Operator

👉 Used for **range (inclusive)**

```sql
SELECT * FROM Products
WHERE Price BETWEEN 10 AND 20;
```

### NOT BETWEEN

```sql
WHERE Price NOT BETWEEN 10 AND 20;
```

---

# 🔹 SQL Aliases (AS)

👉 Temporary name for column/table

## Column Alias

```sql
SELECT CustomerName AS Name FROM Customers;
```

## Table Alias

```sql
SELECT c.CustomerName
FROM Customers AS c;
```

---

# 🔥 Final Quick Revision

* **Aggregate = multiple → single value**
* **COUNT(*) includes NULL, others ignore NULL**
* **LIKE = pattern search (`%`, `_`)**
* **IN = multiple OR shortcut**
* **BETWEEN = range (inclusive)**
* **Alias = temporary name using AS**
